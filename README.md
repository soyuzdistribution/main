## Описание проекта

SOYUZ - это творческое объединение, исследующее новые формы взаимодействия со звуковым контентом. Проект намеренно нарушает привычные паттерны пользовательского интерфейса, создавая хаотичное, нелинейное пространство для взаимодействия с аудиофайлами.

### Основная концепция

Проект исследует возможности деконструкции традиционного интерфейса и создания намеренно хаотичной среды, которая:

1. Нарушает линейность и предсказуемость размещения элементов
2. Создает каждый раз новый визуальный и пользовательский опыт
3. Ставит под вопрос общепринятые паттерны взаимодействия с аудио-контентом

## Структура проекта

Проект состоит из двух основных страниц:

### 1. Главная страница (index.html)

Страница с хаотично расположенными аудиофайлами, позволяющая:
- Просматривать аудиофайлы, расположенные в случайном порядке
- Воспроизводить треки по клику
- Управлять воспроизведением через минималистичный плеер
- Наблюдать визуализацию звука в фоновом режиме

Особенности:
- Случайное расположение аудиофайлов при каждой загрузке страницы
- Случайные цвета и углы поворота для иконок
- Минимальное расстояние между файлами для избежания наложений
- Светлая тема оформления

### 2. Страница блога (blog.html)

Минималистичная страница для публикации статей:
- Фиксированная ширина текста (20% экрана)
- Хронологическое отображение (новые статьи сверху)
- Отображение даты публикации
- Плавная типографика и минималистичный дизайн

## Технические особенности

### Фронтенд

- **HTML5** для структуры
- **CSS3** для стилизации с использованием переменных
- **JavaScript** (чистый JS без фреймворков) для:
  - Динамического создания и размещения аудиофайлов
  - Воспроизведения аудио через Web Audio API
  - Алгоритма предотвращения перекрытия элементов
  - Визуализации звука на холсте (canvas)

### Аудио

- Поддержка форматов MP3, WAV и других стандартных аудиоформатов
- Аудио-визуализация с использованием Web Audio API и Canvas
- Автоматическое определение и отображение названий файлов

### Безопасность

- Отключена возможность загрузки файлов для обычных посетителей
- Файлы добавляются администратором напрямую в директорию `audio/`

## Запуск проекта

Для локального запуска проекта необходим простой веб-сервер:

```bash
# С использованием Python 3
python3 -m http.server 8000

# Или с использованием Node.js
npx serve

# Или с использованием PHP
php -S localhost:8000
```

После запуска сервера проект будет доступен по адресу: `http://localhost:8000`

## Публикация в блоге

### Создание новых постов

Для добавления новых постов в блог:

1. Создайте текстовый файл с расширением `.txt`
2. Имя файла должно быть в формате: `timestamp_Название_поста.txt`, где:
   - `timestamp` - временная метка в миллисекундах (Unix timestamp)
   - `Название_поста` - название поста с подчеркиваниями вместо пробелов

3. Содержимое файла:
   - Первая строка: заголовок поста
   - Вторая строка: пустая
   - Дальше: содержимое поста

4. Загрузите файл в папку `reports/` на сервере

### Пример структуры файла поста:

```
Заголовок поста

Первый абзац с текстом. Этот текст будет отображаться 
как один абзац в посте блога.

Второй абзац. Пустая строка между абзацами создает 
новый параграф в HTML.

Третий абзац с важной информацией.
```

### Дата публикации

- Дата публикации извлекается из временной метки в имени файла
- Формат отображения: ДД.ММ.ГГГГ ЧЧ:ММ:СС.ммм (UTC+1)
- Временную метку можно получить с помощью `Date.now()` в JavaScript или онлайн-конвертеров

### Пример получения временной метки:

В JavaScript:
```javascript
const timestamp = Date.now();
console.log(timestamp); // Выведет что-то вроде: 1711363200000
```

### Отслеживание новых публикаций

Система автоматически проверяет наличие новых постов в папке `reports/` при каждом запуске страницы блога:

1. При первом посещении страницы блога запоминается текущая дата
2. При следующих посещениях система сравнивает временные метки файлов в папке `reports/` с сохраненной датой последнего посещения
3. Если обнаружены файлы с более новыми датами, пользователь получает уведомление о новых публикациях
4. Уведомление автоматически скрывается через 10 секунд или может быть закрыто вручную

Преимущества такого подхода:
- Пользователи всегда знают о появлении нового контента
- Система работает полностью на клиентской стороне и не требует серверных скриптов
- Интеграция с существующим механизмом публикации через txt-файлы
